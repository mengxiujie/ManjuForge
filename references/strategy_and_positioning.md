# ManjuForge 定位与推广策略

## 核心定位

**Slogan**：别人做好看的单集，我们做活得久的IP。

**赛道**：漫剧长线连载稳定性（避开短视频画质红海）

**差异化**：市面AI漫剧工具都在卷单集成片/画面渲染，没有人解决"连载30集以上人设崩坏"的痛点。

## 执行策略（用户已确认）

```
第一步：Hermes Skill版 → 用Demo和实际用户验证市场
第二步：观察Star/Issue/私信询盘
第三步：有需求信号后 → 启动Flask企业版
```

关键原则：知识库（拉片案例、模板、规则）是真正的壁垒，不是Flask代码。
Skill版和Flask版共享同一套知识库，不存在浪费。

## 技术壁垒诚实评估

> 来源：第三方独立评估

当前方案本质是 Character Card + Summary Memory + Story Bible + Prompt Injection 的组合。
这是**工程壁垒**，不是**技术壁垒**。可落地，但不属于难复制技术。

对外宣传应诚实：
- ❌ "自研记忆引擎" → ✅ "Prompt级记忆系统"
- ❌ "彻底解决人设崩坏" → ✅ "显著降低长线连载中的人设漂移与剧情断层"
- ❌ "国内首个" → ✅ 用Demo数据说话（"3集实测：伏笔回收率100%，人设零崩坏"）

## 长期战略方向

从"AI漫剧工具"升级为"AI叙事基础设施"（AI Narrative Infrastructure）。
漫剧只是应用场景之一，同一套底层可扩展到：网文、漫画、RPG、Galgame、互动小说、VTuber、AI陪伴。

### 6个月聚焦策略

只做三件事：
1. **Story Bible** — 解决"记不住设定"
2. **Character Graph** — 解决"记不住关系"
3. **Foreshadow Engine** — 解决"记不住伏笔"

❌ Agent、❌ Web美化、❌ SaaS平台、❌ AI视频、❌ AI绘图，全部暂停。

详见 `references/third_party_evaluation.md`。
