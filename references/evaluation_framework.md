# 漫剧评估框架

> 基于《代码赘婿》Demo评估实践总结的六维度评估体系。每集创作后逐项打分(1-10)，综合加权得出总分。

---

## 评估维度与权重

| 维度 | 权重 | 核心关注 |
|------|:----:|----------|
| 剧本 | 20% | 结构、台词、人物、伏笔 |
| 分镜 | 20% | 碎片节奏、运镜、视觉锤、时间轴 |
| 配音 | 15% | 台词精准度、情绪标注、平台适配 |
| AI提示词 | 15% | 风格锚点、Character ID、负向提示词、跨镜头一致性 |
| 记忆机制 | 15% | 跨集连贯性、伏笔追踪、人物状态准确性 |
| 抖音适配 | 15% | 标题、钩子、互动引导、时长、算法适配 |

## 跨集评估（每3集执行一次）

除了每集的六维度自检外，每3集应执行一次跨集评估，额外检查：

| 检查项 | 说明 |
|--------|------|
| State Snapshot连续性 | 对比3集的State Snapshot，验证状态是否在推进 |
| 关系演进节奏 | 关系值是否过快/过慢（参考 `state_snapshot_guide.md`） |
| Faction Conflict升级 | 故事是否从Character Conflict向Faction Conflict演进 |
| 问题解决进度 | 主角面对的问题是否在逐步解决（非原地踏步） |
| 第二条主线进度 | 是否有至少2条主线在同时推进 |

详细State Snapshot方法见 `references/state_snapshot_guide.md`。

## 各维度评估标准

### 剧本（10分制）

| 分数 | 标准 |
|------|------|
| 9-10 | 台词精准立人物，伏笔自然不刻意，情绪曲线有层次 |
| 7-8 | 结构完整，台词合格，有1-2处可优化 |
| 5-6 | 结构松散或台词平淡，有明显逻辑问题 |
| 3-4 | 人物行为不符合设定，剧情有硬伤 |
| 1-2 | 基本不可用 |

**高频扣分项**：
- 闪回时机打断情绪递进
- 主角台词过少（存在感弱）
- 对话太直白（像旁白解说而非角色说话）
- 女主/重要角色未出场

### 分镜（10分制）

| 分数 | 标准 |
|------|------|
| 9-10 | 碎片节奏精准(3-7秒)，运镜丰富，有视觉锤，时间轴连贯 |
| 7-8 | 节奏合格，有1-2处碎片偏长或运镜单一 |
| 5-6 | 碎片偏长(>10秒平均)，运镜几乎全是"固定"，缺视觉锤 |
| 3-4 | 碎片混乱，时间轴有错误，景别无节奏 |
| 1-2 | 基本不可用 |

**高频扣分项**：
- 碎片平均时长>10秒（抖音节奏太慢）
- 运镜几乎全是"固定"（缺乏视觉变化）
- 缺少视觉锤镜头（没有让观众截图的瞬间）
- 跨时段无过渡碎片
- 重复信息的P2碎片未砍

### 配音（10分制）

| 分数 | 标准 |
|------|------|
| 9-10 | 台词精准展示人物性格，情绪标注完整，平台适配正确 |
| 7-8 | 台词合格，有1-2处可优化 |
| 5-6 | 台词平淡或不符合人物，平台适配有误 |
| 3-4 | 台词严重偏离人设 |
| 1-2 | 基本不可用 |

**高频扣分项**：
- 旁白多余（抖音古风短剧主流是纯字幕+台词）
- 主角台词过少
- 缺少情绪标注

### AI提示词（10分制）

| 分数 | 标准 |
|------|------|
| 9-10 | 统一风格锚点，Character ID完整，负向提示词统一，跨镜头一致 |
| 7-8 | 有风格锚点但个别遗漏，Character ID基本完整 |
| 5-6 | 缺风格锚点或Character ID，跨镜头一致性差 |
| 3-4 | 无锚点无ID，画风会崩 |
| 1-2 | 基本不可用 |

**高频扣分项**：
- 🔴 无风格锚点（跨镜头画风崩=致命问题）
- 无Character ID（角色跨集变脸）
- 无负向提示词（AI会画出现代元素/水印）
- 声音设计与场景时间不匹配（清晨写"烛火"）

### 记忆机制（10分制）

| 分数 | 标准 |
|------|------|
| 9-10 | 全部记忆元素延续，伏笔追踪准确，人物状态无bug |
| 7-8 | 大部分延续，有1处小bug |
| 5-6 | 部分延续但有明显遗漏或错误 |
| 3-4 | 记忆断裂，多处矛盾 |
| 1-2 | 基本不可用 |

**高频扣分项**：
- 🔴 角色出场状态bug（"门口一闪"≠"正式出场"）
- 标志性习惯未延续
- 伏笔遗忘
- 口头禅每集都用（应该克制，留给关键场景）

### 抖音适配（10分制）

| 分数 | 标准 |
|------|------|
| 9-10 | 标题悬念强且兑现，有互动+关注引导，时长合适，无剧透 |
| 7-8 | 标题合格，引导基本到位，有1处可优化 |
| 5-6 | 标题平淡或有小问题，引导缺失 |
| 3-4 | 标题欺诈或严重剧透 |
| 1-2 | 基本不可用 |

**高频扣分项**：
- 🔴 标题欺诈（引用剧中未出现的台词=划走+取关）
- 关注引导剧透下集核心事件
- 互动引导是二选一（应改为开放式问题）
- 时长<2分30秒（被算法判定为"短视频"而非"短剧"）

```markdown

| 维度 | 上集 | 本集 | 变化 |
|------|:----:|:----:|:----:|
| 剧本 | X.X | X.X | +X.X |
| 分镜 | X.X | X.X | +X.X |
| 配音 | X.X | X.X | +X.X |
| AI提示词 | X.X | X.X | +X.X |
| 记忆机制 | X.X | X.X | +X.X |
| 抖音适配 | X.X | X.X | +X.X |
| **综合** | **X.X** | **X.X** | **+X.X** |

### 优先修复清单
1. 🔴 （致命问题）
2. 🟡 （重要问题）
3. 🟢 （优化项）
```

---

## 评估-修复闭环

```
创作 → 评估（六维度打分）→ 修复（按优先级）→ 更新记忆文件 → 下一集
```

每集评估后：
1. 按🔴→🟡→🟢顺序修复
2. 将发现的新坑同步patch到SKILL.md的自检清单
3. 更新剧情摘要
4. 再开始下一集
