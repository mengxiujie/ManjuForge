# State Snapshot 格式

> 来源：第三方评估中提出的剧情状态追踪方法。
> 用途：每集创作完成后，用此格式记录世界状态变化，验证剧情是否在"状态推进"而非"原地踏步"。

---

## 什么是State Snapshot

State Snapshot是将剧情从"文字描述"转化为"可量化状态"的方法。
它不是用来限制创作，而是用来**验证**剧情是否在真正推进。

---

## 格式模板

```
Episode X

主要角色:
  角色A:
    信任: [0-100]
    地位: [上升/稳定/下降]
    状态: [关键词]

  角色B:
    敌意: [0-100]
    策略: [关键词]

世界状态:
  组织A:
    状态: [关键词]
  组织B:
    状态: [关键词]

主线进度:
  主线A: [百分比]%
  主线B: [百分比]%
  主线C: [百分比]%

新增状态:
  [本集新增的世界状态变化]

新增资源/冲突:
  [本集新增的资源或冲突类型]
```

---

## 使用规则

### 1. 每集完成后填写
创作完一集后，花2分钟填写State Snapshot，对比上一集的变化。

### 2. 验证"状态推进"
检查：每个角色的状态是否有变化？如果某个角色连续2集状态不变，说明他/她的剧情线停滞了。

### 3. 验证"问题逐步解决"
检查：主角面对的问题是否在逐步解决（而非永远有新危机）？
- ✅ 问题→解决一部分→面对新问题
- ❌ 危机→新危机→再新危机（永远不解决）

### 4. 验证"Faction Conflict升级"
检查（第6集左右）：故事是否从个人对抗升级为组织/阵营对抗？
- Character Conflict: A vs B
- Faction Conflict: 组织A vs 组织B vs 组织C

### 5. 关系值节奏参考
10集长篇的关系演进节奏建议：
- E1-E3: 快速推进（每集+10~15）
- E4-E6: 稳定推进（每集+10）
- E7-E10: 平台期（每集+5，留空间给后续）

## 示例（《代码赘婿》第5集）

```
Episode 5

沈舟:
  信心: 65
  地位: 上升

林霜:
  信任: 60
  尊重: 55

赵天赐:
  敌意: 95

林家:
  现金流: 回升（18匹订单）

主线A（账本案）: 30%（已揭露，但贪墨者未抓）
主线B（林父往事）: 20%（父亲遗训被引用）
主线C（穿越秘密）: 5%（仅"以前在别的地方学的"）
主线D（原料来源）: 0% → 激活（赵天赐开始查）

新增状态: 陈府成为林家第一个B端客户
新增资源: 街口摊位（临时销售渠道）
新增冲突: 原料来源被调查
```

---

## 注意事项

- State Snapshot是**验证工具**，不是**创作约束**。先写故事，再填Snapshot
- 不需要精确到个位数，大致范围即可
- 如果某集Snapshot显示"所有状态都没变"，说明这集需要重写
