# ManjuForge 漫剧铸造 - AI漫剧连载创作引擎

## 定位

国内首个聚焦「漫剧长线连载稳定性」的轻量化开源创作方案。

核心差异化：别人做好看的单集，我们做活得久的IP。

## 适用场景

- 用户要从零创作一部漫剧（输入题材 → 全套产出）
- 用户要把小说/网文改编为漫剧剧本
- 用户在做连载漫剧，需要跨集保持人设和剧情连贯
- 用户需要竖屏9:16分镜（面向AI生成器）
- 用户需要批量生成漫剧内容

## 核心产出（八大模块）

每次完整创作产出以下内容：

1. **人设档案** - 角色外貌/性格/关系/台词风格/成长弧线
2. **世界观设定** - 时代/地理/规则/势力/核心矛盾
3. **剧情大纲** - 全剧主线+分集梗概
4. **开篇钩子** - 前3秒视觉锚点+悬念设计
5. **逐镜分镜** - 横屏16:9标准版 或 竖屏9:16 AI执行版
6. **情绪标注** - 每幕情绪曲线+节奏节点
7. **配音文案** - 适配短视频的台词/旁白/独白
8. **AI绘图提示词** - 每幕配套构图/光影/风格提示词

## 一、连载记忆机制（核心壁垒）

### 原理

用文件系统实现轻量化持久记忆。每个漫剧项目维护三个记忆文件：

| 文件 | 作用 | 更新时机 |
|------|------|----------|
| `角色档案.md` | 锁定人设、关系、台词风格 | 每次创作前读取，人设变更时更新 |
| `世界观.md` | 锁定时代、规则、势力 | 首次创作时写入，世界观扩展时更新 |
| `剧情摘要.md` | 记录已发生的关键剧情、伏笔、时间线 | 每集创作完成后自动追加 |

### 记忆工作流

```
第1集：创建角色档案 + 世界观 → 生成剧本+分镜 → 写入剧情摘要
第2集：读取三个记忆文件 → 基于约束生成 → 更新剧情摘要
第3集：读取三个记忆文件 → 基于约束生成 → 更新剧情摘要
...
第N集：始终锚定历史记忆，杜绝人设OOC
```

### 记忆约束规则（生成时强制遵守）

生成新一集内容时，必须在Prompt中注入以下约束：

```
【角色锁定】
{角色档案.md 的完整内容}

【世界观锁定】
{世界观.md 的完整内容}

【前情提要】
{剧情摘要.md 的最近5集摘要}

【硬性规则】
- 角色性格、说话风格、关系不得与档案矛盾
- 已死角色不可复活（除非世界观允许）
- 已揭露的伏笔必须有回收或明确延续
- 时间线不得跳跃或矛盾
- 新伏笔需在摘要中记录
```

### 视觉符号跨集进化技巧

视觉符号（道具、动作、色调）应跨集延续但每集升级，形成"情绪刻度尺"：

| 集数 | 示例：家主令牌 | 示例：敲桌子 | 示例：bug口头禅 |
|------|---------------|-------------|----------------|
| 第1集 | 沈舟目光追着令牌（注意到） | 桌面上敲（无意识） | "这个bug该修了"（私下宣言） |
| 第2集 | 林霜攥紧指节发白（紧张） | 膝盖上敲（焦虑） | 不出现（克制） |
| 第3集 | 站起来说话时可见（权威） | 屏风边缘敲（紧张等待） | 当众说（态度升级） |
| 第4集 | 转身时晃动（陪伴） | 门框+腰间+桌面（三个变体） | 自嘲（"这算什么bug？"） |
| 第5集 | 林霜主动不戴（放下身份） | 未出现（合理缺失） | 未出现（合理缺失） |

**规则**：
- 同一符号每集出现但变换表达方式，避免重复感
- 口头禅/标志性台词留给关键场景，不要每集都用
- 符号"缺席"也是叙事——当角色不再需要某个符号时，说明成长了

### 空间距离叙事法（已验证）

角色之间的物理距离可以隐喻关系变化。以《代码赘婿》林霜为例：

| 集数 | 空间位置 | 关键动作 | 情感阶段 |
|------|----------|----------|----------|
| 第1集 | 门外 | 冷眼扫过 | 无视 |
| 第2集 | 门口 | 没接纸但自己验证 | 好奇 |
| 第3集 | 偏院→正厅 | 站起来宣布 | 正眼 |
| 第4集 | 面前 | 主动走到面前求助 | 并肩 |
| 第5集 | 身边 | 并肩走路+给面子+送汤 | 关心 |

**规则**：让角色的物理距离逐集缩短（或拉远），比直接写"她开始喜欢他"有力得多。

### 高级人物塑造："行为否定+心理认可"

> 来源：第三方评估，以第2集林霜"没接纸但回房翻账本"为例

当角色的**行为**和**心理**方向相反时，人物最立体：
- 林霜没接纸 = 行为否定（"我是家主，不能轻信赘婿"）
- 林霜翻账本 = 心理认可（"但他说的可能是真的"）

**规则**：不要让角色直接说"我相信你"。让她用一个否定行为+一个肯定行为同时表达，观众自己会读懂。

### 第二条主线引入时机（第4集起）

> 来源：第三方评估——"前三集全部围绕账本运转，到第8集观众会疲劳"

**规则**：从第4集开始必须引入第二条主线。第一条主线解决"外部冲突"，第二条主线探索"角色内心"。

两条主线的交织方式：
- 第4集：第二条线的种子（如林霜提到"父亲旧交"）
- 第5-6集：第二条线展开（如林霜父亲的故事）
- 第7-8集：两条线交汇（如父亲的遗训帮助解决当前危机）

### 信任曲线节奏控制（6集实测+第三方验证）

> 来源：第三方评估——"林霜信任增长略快，已接近恋爱前夜"

角色之间的信任/情感不能线性增长到100%。必须在中段进入平台期：

| 集数 | 信任值 | 阶段 |
|------|:------:|------|
| E1-3 | 0→30 | 冷漠→认可 |
| E4-5 | 45→60 | 给机会→维护尊严 |
| E6 | 65 | 托付 |
| E7-8 | 70-75 | **平台期：共同战斗** |
| E9-10 | 75-80 | 缓升：依赖+默契 |

**关键规则**：第6集后不能再"更关心/更亲近/更暧昧"，必须进入"共同战斗+共同承担风险"阶段。信任到80%后需要一个"考验/危机"才能突破。

### 势力对抗升级时机（第6集起）

> 来源：第三方评估——"第6集从Character Conflict进入Faction Conflict"

| 阶段 | 冲突类型 | 集数范围 |
|------|----------|----------|
| 个人冲突 | 主角vs反派（打脸） | E1-3 |
| 商业自救 | 主角vs经济危机 | E4-5 |
| 势力对抗 | 联盟vs垄断vs行会 | E6+ |

**规则**：到第7-8集如果还是"个人打脸"模式，观众会疲劳。必须引入组织/资源/供应链等系统性对抗。

### 联盟/合作形成节奏

> 来源：第三方评估——"联营成功太快，实际商业里小商户最怕风险"

组建联盟时不能一步到位。更合理的节奏：
```
10家 → 1家愿意 → 8家观望 → 1家反对
     → 断供危机爆发
     → 联盟在危机中真正形成
```

**规则**：合作方加入必须有阻力（利益顾虑/害怕报复/内部反对），不能"一句话说服所有人"。

### 批量创作模式

### 剧情摘要更新模板

每集创作完成后，必须按以下模板更新剧情摘要：

```markdown
### 第X集：{悬念句标题}

**核心事件**：
- （3-5个要点）

**情绪曲线**：（开头→中段→结尾的情绪走向关键词）

**新伏笔**：
- （列出本集新埋的伏笔）

**人物变化**：
- （每个出场角色的状态变化）

**结尾悬念**：（一句话）
```

## 二、创作流程

### 阶段一：项目初始化（首次创作时）

当用户提出新漫剧创作需求时：

**1. 信息收集**（至少需要以下信息，缺失的主动询问）
- 题材/类型（都市/古风/悬疑/甜宠/玄幻/校园等）
- 目标平台（抖音/小红书/快手/视频号）
- 画幅（横屏16:9 或 竖屏9:16）
- 目标集数
- 核心卖点/一句话概念

**标题规范**：每集标题必须是悬念句或冲突句，禁止"第X集：XXX"格式。
- ✅ "赘婿被当众扇耳光，他忍了，但账本上的数字让他笑了"
- ❌ "第1集：穿越"

**2. 生成人设档案**
- 基于题材和用户描述，生成3-5个核心角色
- 每个角色包含：外貌、性格、关系网、台词风格、成长弧线
- 输出为 `角色档案.md`

**3. 生成世界观**
- 基于题材生成时代背景、核心规则、势力关系、核心矛盾
- 输出为 `世界观.md`

**4. 生成剧情大纲**
- 全剧主线脉络
- 分集梗概（每集1-2句话）
- 关键转折点和高潮标注

**5. 用户确认**
- 将人设、世界观、大纲展示给用户确认
- 根据反馈调整后再进入剧本生成

### 阶段二：单集创作（标准四步工作流）

每集创作严格执行：**创建 → 评估 → 修复 → 同步**

**Step 1：创建**
1. 读取记忆（角色档案+世界观+剧情摘要）
2. 生成完整版：剧本+分镜+配音+AI提示词+质量自检
3. 输出为 `第X集_完整版.md`

**Step 2：评估**
按六维度打分（见 `references/evaluation_framework.md`）：
- 剧本（结构/台词/人物/伏笔）
- 分镜（碎片节奏/运镜/视觉锤）
- 配音（台词精准度/情绪标注）
- AI提示词（风格锚点/Character ID/一致性）
- 记忆机制（跨集连贯性/伏笔追踪）
- 抖音适配（标题/钩子/互动引导/时长）
列出所有需要修复的问题，按🔴🟡🟢分级。

**Step 3：修复**
逐一执行评估中发现的修复项。修复后更新质量自检。

**Step 4：同步**
- 更新剧情摘要（核心事件+情绪曲线+新伏笔+人物变化+结尾悬念）
- 同步文件到桌面文件夹和仓库demo/目录

**1. 读取记忆**
- 加载 `角色档案.md`
- 加载 `世界观.md`
- 加载 `剧情摘要.md`（如存在）

**2. 生成剧本**
- 基于大纲中该集的梗概 + 记忆约束
- 结构：开篇钩子（前3秒）→ 冲突建立 → 情绪递进 → 转折/反转 → 悬念截断结尾
- 短视频节奏：3秒留人、中段强冲突、结尾留悬念
- **开篇钩子必须第一帧有画面，禁止黑屏开头**（抖音用户划得快，黑屏=划走）
- **每集结尾必须有**：互动引导（"评论区告诉我..."）+ 关注引导（"关注看下一集..."）
- **重要角色首次出场时加字幕卡**（姓名+身份，如"赵天赐·赵家嫡子"）

**3. 生成分镜**
- 根据用户选择的画幅模式生成：
  - 横屏标准版：镜号/场景/景别/运镜/时长/画面描述/台词/情绪/色调
  - 竖屏AI执行版：碎片编号/时间轴/景别运镜/画面描述/声音四层/AI提示词/优先级

**4. 生成配音文案和AI提示词**
- 配音文案：适配短视频的台词+旁白+独白
- AI提示词：每幕配套构图/光影/风格提示词

**5. 更新记忆**
- 将本集关键剧情、新伏笔、人物变化追加到 `剧情摘要.md`
- **更新后自动触发质量自检**（见第六节清单）
- **每3集执行一次六维度评估**（见 `references/evaluation_framework.md`），评分低于7分的维度必须修复后才能继续下一集
- **必须立即执行质量自检**（见第六节），不要等用户要求
- **必须执行六维度评估**（见 `references/evaluation_framework.md`），每集打分并记录趋势

### 阶段三：连续续写

后续每集重复阶段二流程，但：
- 始终读取最新记忆文件
- 剧情摘要自动滚动（保留最近N集详细摘要，更早的压缩为一句话）
- 用户可随时修改角色档案或世界观（修改后立即生效）

## 三、风格模板

内置六种漫剧风格，用户选择后自动调整生成策略。每种风格的详细调整规则（台词/分镜/色调/节奏/AI提示词锚点）见 `references/style_guide.md`。

| 风格 | 节奏 | 情绪基调 | 核心公式 |
|------|------|----------|----------|
| 甜宠 | 轻快、甜蜜 | 温暖、心动 | 误会→解开心结→甜蜜互动 |
| 虐恋 | 压抑、隐忍 | 痛苦、遗憾 | 误会加深→牺牲→隐忍→爆发 |
| 悬疑 | 紧凑、层层递进 | 紧张、不安 | 伏笔→误导→反转→真相 |
| 校园 | 轻松、真实 | 青春、青涩 | 日常→暗恋→表白→成长 |
| 古风 | 舒缓、雅致 | 古韵、唯美 | 权谋→情感→家国→抉择 |
| 玄幻 | 快节奏、高燃 | 热血、震撼 | 修炼→战斗→突破→逆袭 |

**风格切换时必须调整**：AI提示词锚点、台词风格、分镜景别比例、色调方案、视觉锤选择。

### 风格对生成策略的具体影响

选择风格后，以下参数必须调整：

| 参数 | 甜宠 | 虐恋 | 悬疑 | 校园 | 古风 | 玄幻 |
|------|------|------|------|------|------|------|
| 台词密度 | 高（多对话互动） | 中（多内心独白） | 低（留白制造不安） | 高（日常化） | 中（古韵对白） | 低（画面驱动） |
| 碎片时长 | 3-5秒（轻快切换） | 5-8秒（留情绪空间） | 3-5秒（紧凑推进） | 5-7秒（自然节奏） | 6-9秒（舒缓留白） | 3-5秒（高燃快切） |
| 声音基调 | 🎵钢琴/吉他·轻快 | 🎵大提琴·低沉 | 🎵弦乐·不和谐 | 🎵尤克里里·清新 | 🎵古琴/箫·悠扬 | 🎵鼓点/电子·燃 |
| 情绪曲线 | 甜→小虐→更甜 | 虐→更虐→爆发 | 疑→误导→反转 | 日常→暗涌→表白 | 权→情→抉择 | 苦→战→逆袭 |
| 结尾类型 | 甜蜜悬念（他看到她了） | 刀子悬念（她转身走了） | 真相悬念（门后面是……） | 心动悬念（他/她说了什么） | 命运悬念（圣旨到了） | 燃点悬念（新敌人出现） |
| 视觉色调 | 暖粉/浅金 | 冷灰/深蓝 | 暗绿/冷白 | 明亮/自然光 | 暖金/墨色 | 高对比/火焰色 |

## 四、分镜规范

### 横屏标准版（16:9）- 面向人类摄制组

必含字段：镜号、场景、景别、运镜、时长、画面描述、台词/音效、情绪、色调

详细规范参考 `anime-storyboard-artist` Skill 的横屏模式。

### 竖屏AI执行版（9:16）- 面向AI生成器

必含字段：碎片编号、段落、时间轴、景别/运镜、画面描述、声音四层设计、AI提示词/优先级

关键规则：
- 前3秒必须有视觉钩子（大特写或强对比画面）
- 碎片时长10-15秒，总时长≤5分钟
- 声音四层标注：🎵音乐层 + 🔊环境层 + 💓生理层 + 🫧细节层
- **每碎片必须有统一的风格锚点（Style Anchor）前缀**，确保跨镜头画风一致
- **每个角色必须有Character ID锚定**（如`Character ID: SHENZHOU`），跨集复用
- **所有提示词必须包含负向提示词**：NO text, NO watermark, NO modern elements
- 悬念截断式结尾

详细规范参考 `anime-storyboard-artist` Skill 的竖屏模式及以下知识库文件：
- `references/knowledge_base/vertical_short_drama.md` - 竖屏专项
- `references/knowledge_base/ai_execution_fields.md` - AI执行字段
- `references/knowledge_base/sound_design_layers.md` - 声音四层

### 分镜踩坑要点

- **平均碎片时长3-7秒**——抖音爆款节奏是快切，超过10秒/碎片会让观众走神
- **砍P2碎片给关键碎片让时间**——重复信息的碎片（如已展示过的嘲笑再给特写）必须砍掉
- **关键碎片加运镜变化**——不能全是"固定"，至少在出场/对峙/情绪转折碎片加"缓慢推进""正反打""缓慢后拉"
- **跨时段必须有过渡碎片**——清晨→午后→夜晚，用空镜（日影移动、烛火点燃）过渡
- **声音设计必须与场景时间匹配**——清晨=鸟鸣+晨光，午后=蝉鸣+阳光，夜晚=烛火+虫鸣

### 长篇连载踩坑要点（6集实测经验）

- **🔴 视觉符号必须跨集进化**——令牌：目光追→攥紧→站起可见→主动不戴。敲桌子：桌面→膝盖→屏风→门框→腰间→椅子背。同一符号每集出现但变换表达方式
- **🔴 口头禅/标志性台词不要每集都用**——留给关键场景。"bug"：第1集私下→第3集当众→第4集自嘲→第5-6集不用。克制使用才有力量
- **🔴 连续"温情/建设"不超过2集**——抖音观众需要爽感。第4-5集都是商业建设，第6集必须回归冲突/打脸
- **🔴 第二条主线应在第4集前启动**——单一主线（如账本）到第8集会疲劳。第4集起埋入新主线（如林霜父亲的故事）
- **🔴 第6集起必须升级为势力对抗**——个人打脸模式到第5集为止，第6集起引入组织/资源/供应链对抗
- **🔴 信任曲线第6集后必须进入平台期**——不能再"更关心/更亲近"，应进入"共同战斗"阶段
- **🔴 联盟形成不能一步到位**——必须有阻力（1愿意/8观望/1反对），危机中才真正形成
- **🔴 D段（反派反应）要简洁**——反派场景3个碎片足够（汇报→反应→决定），不要拖到4-5个碎片
- **🟡 "行为否定+心理认可"是高级人物塑造**——林霜没接纸但自己验证、没送汤但问更深的问题。表面拒绝+内心接受=立体人物
- **🟡 空间距离是隐性叙事工具**——林霜：门外→门口→偏院→面前→并肩→身边。不需要台词，距离本身就讲故事

### 🔴 实测踩坑记录（5集Demo验证）

### 人设一致性
- [ ] 角色性格与档案一致
- [ ] 台词风格与档案一致（口头禅、说话节奏）
- [ ] 人物关系无矛盾
- [ ] 角色外貌描述无偏差
- [ ] 🔴 角色出场状态准确——"仅被提及"≠"出场"，"门口一闪而过"≠"正式出场"，剧情摘要中必须区分

### 剧情逻辑
- [ ] 时间线连贯无跳跃
- [ ] 与前集剧情衔接自然
- [ ] 伏笔有延续或回收
- [ ] 无已死角色复活等逻辑漏洞
- [ ] 🔴 时间跳跃有标记——跨时段（清晨→午后→夜晚）必须有空镜或字幕卡过渡

### 短视频节奏
- [ ] 🔴 前3秒有视觉钩子——第一帧必须有画面，禁止黑屏开头（抖音用户划得快，黑屏=划走）
- [ ] 中段有冲突/反转
- [ ] 结尾有悬念截断
- [ ] 碎片时长符合规范（竖屏模式，平均3-7秒/碎片）
- [ ] 有"视觉锤"镜头——让观众截图的高光瞬间（如攥令牌指节发白、烛光侧脸定格）
- [ ] 有角色字幕卡——重要角色首次出场时标注姓名+身份
- [ ] 结尾有互动引导——用开放式问题（"如果是你，你会怎么做？"），不用二选一
- [ ] 结尾有关注引导——不剧透下集核心事件，用悬念句（"林家的水比账本更深"）
- [ ] 🔴 P2碎片可砍——重复信息的碎片（如已展示过的嘲笑再给特写）应删除，时间分给关键碎片
- [ ] 🔴 问题必须逐步解决——不能永远"危机→新危机→再新危机"，要让主角"解决一部分"再面对新问题（状态推进，而非原地踏步）
- [ ] 🔴 群体不能一呼百应——涉及联盟/说服/拉拢场景，必须有"1个愿意/多数观望/少数反对"的分层，否则不真实

### AI绘图一致性
- [ ] 🔴 所有提示词有统一风格锚点（Style Anchor）——没有锚点=跨镜头画风崩
- [ ] 🔴 所有角色有Character ID锚定（如`Character ID: SHENZHOU`）——跨集复用同一ID
- [ ] 所有场景有Scene ID（如`Scene ID: HALL`）+色值定义
- [ ] 所有提示词包含负向提示词（NO text, NO watermark, NO modern elements）
- [ ] 同一角色跨集视觉一致（复用相同Character ID+描述）
- [ ] 🔴 声音设计与场景时间一致——清晨场景不能写"烛火噼啪"，夜晚场景不能写"鸟鸣"

### 抖音平台适配
- [ ] 🔴 标题必须是悬念句或冲突句，且内容必须在本集中兑现——引用剧中未出现的台词=标题欺诈=划走+取关
- [ ] 时长3-5分钟（太短被算法判定为"短视频"而非"短剧"，推荐权重不同）
- [ ] 关注引导不剧透下集核心事件
- [ ] 🔴 反派离场前必须有台词——沉默离场缺乏张力，一句冷笑/狠话让观众记住
- [ ] 🔴 大型场景（议亲/宴会/审判）的物理合理性——角色从角落走到中央需要时间，不能凭空出现在婚书旁
- [ ] 🔴 台词量大的碎片必须拆分——算账/辩论/长独白超过8秒必须拆成2个碎片（提问+回答）

### 记忆连贯性（跨集专项）
- [ ] 读取角色档案+世界观+剧情摘要后再开始创作
- [ ] 视觉符号跨集延续但有变化——如令牌：第1集目光追→第2集攥紧→第3集……（每集升级）
- [ ] 标志性习惯每集出现但变换场景——如敲桌子：桌面→膝盖→扶手上
- [ ] 口头禅/标志性台词不要每集都用，留给关键场景
- [ ] 🔴 创作完成后必须更新剧情摘要——包括：核心事件、情绪曲线、新伏笔、人物变化、结尾悬念
- [ ] 人物状态快照中"最后出场"必须准确反映实际出场情况
- [ ] 🔴 关系演进必须控制节奏——10集的剧，第6集就到"恋爱前夜"太快。建议：前6集快速推进，第7集起进入平台期（共同战斗/共同承担风险），留空间给后续。具体节奏参考：信任值从0开始，每集+10~15，到60左右进入平台期
- [ ] 关系表达优先用"Show Not Tell"——用空间距离变化（3步→0步）、道具传递（送汤）、行为转变（不叫"赘婿"叫"管事的"）代替台词直说"我相信你了"
- [ ] 🔴 长篇剧（10集+）必须在中段从Character Conflict升级到Faction Conflict——个人打脸观众会疲劳，势力对抗不会。第6集左右应出现组织/联盟/阵营层面的对抗

## 七、批量创作模式

### 单集创作→评估→修复的标准工作流

每集必须按以下流程执行，不能跳过：

```
1. 读取记忆文件（角色档案+世界观+剧情摘要）
2. 生成完整内容（剧本+分镜+配音+AI提示词+自检）
3. 更新剧情摘要（核心事件+情绪曲线+新伏笔+人物变化+结尾悬念）
4. 同步文件到项目目录
5. 六维度评估（每3集或用户要求时）
6. 修复评估发现的问题
7. 同步修复后的文件
```

**关键**：步骤2完成后必须立即执行步骤3，不能跳到下一集。记忆断裂是连载最大的敌人。

当用户需要批量生成多集时：

**操作步骤：**
1. 一次性读取全部记忆文件（角色档案+世界观+剧情摘要）
2. 按大纲逐集生成，每集完成后：
   - 更新剧情摘要（核心事件+情绪曲线+新伏笔+人物变化+结尾悬念）
   - 执行质量自检清单
3. 每3集做一次六维度评估（见 `references/evaluation_framework.md`）
4. 每5集做一次人设一致性大检查（对比角色档案，逐条验证）
5. 输出格式：每集一个独立文件，命名 `第X集_完整版.md`

**批量模式注意事项：**
- 不能一次性生成所有集——必须逐集生成、逐集更新记忆，否则记忆断裂
- 如果某集评估分数低于7分，必须修复后再继续
- 大纲中的分集梗概是约束，不能偏离；但细节可以灵活发挥
- 如果用户要求跳过某集直接写后面的，必须先补全跳过集的剧情摘要

## 八、Demo驱动迭代工作流（推荐）

每次创作新项目时，建议按以下流程迭代：

```
第1集：创作 → 专业评估（六维度打分）→ 修复全部🔴🟡项 → 修订版
第2集：创作 → 专业评估 → 修复 → 修订版
第3集：创作 → 专业评估 → 修复 → 修订版
每3集：汇总评估趋势，检查Skill本身是否需要更新
```

**为什么这样做**：
- 第1集通常分会最低（6-8分），因为规则还不够具体
- 每次评估发现的问题都应反哺到SKILL.md的自检清单或踩坑要点
- 3集后Skill趋于稳定，后续集数的分数会自然上升
- 这个过程本身就是Skill的"训练"——踩坑越多，规则越完善

**评估后必须做的**：
1. 逐项修复所有🔴🟡问题
2. 将新发现的踩坑点反哺到SKILL.md
3. 更新剧情摘要
4. 同步修复后的文件到仓库副本
